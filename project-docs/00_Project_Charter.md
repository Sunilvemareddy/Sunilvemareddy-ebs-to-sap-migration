# Project Charter — Oracle EBS → SAP Migration

**Project name:** Oracle EBS → SAP Migration (Acquired Business Consolidation)  
**Prepared by:** Sunil — Program Manager  
**Version:** 0.1 (Draft)

## 1. Executive summary
Migrate acquired company's Oracle EBS footprint into corporate SAP environment to standardize operations, reduce maintenance overhead, and consolidate financial and operational reporting.

## 2. Objectives
- Consolidate systems into corporate SAP.
- Preserve master & transactional integrity.
- Rationalize customizations and adopt SAP standard where feasible.

## 3. Scope (high-level)
Modules in scope: FI/CO, MM, SD, PP, WM, QM, PM, HR (as applicable).  
Data scope: Master data, open transactions, defined historical summary.

## 4. Deliverables
Project charter, governance docs, source assessment, data mapping templates, test plans, cutover runbook, hypercare plan.

## 5. Governance & roles
- Executive Sponsor: [Name]
- Program Manager: Sunil
- Steering Committee: CIO, CFO, BU Heads
- Workstream Leads: Data, Functional, Technical, Testing, Change Mgmt

## 6. High-level timeline
Phased plan: Discover → Design → Build → Test → Cutover → Stabilize.

## 7. Risks & assumptions
- Risk: High number of EBS customizations. Mitigation: thorough discovery and rationalization.
- Assumption: Corporate SAP instance available for onboarding.

## 8. Success criteria
- Business sign-off on data reconciliation
- Minimal operational downtime during cutover
- Knowledge transfer complete to in-house team
